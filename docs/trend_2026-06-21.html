# SoC 策略分析報告｜2026-04-20 當週 --- ## 任務一：結構性機會偵測 --- ### 機會 1：邊緣端 LLM 推論加速 SoC——針對「企業內部 AI Agent 私有化部署」的記憶體頻寬與低功耗推論晶片 #### Step 1：事實觀察 - [9to5Google, 2026-04] [來源]「Google 向 Anthropic 投資最高 400 億美元」→ 觀察事實：雲端巨頭加倍押注 LLM 模型供應，模型能力供給端競爭白熱化。 - [Bloomberg / iThome, 2026-06-15] [來源]「美國政府要求 Anthropic 封鎖外國用戶存取 Claude Fable/Mythos，Anthropic 全面暫停提供」→ 觀察事實：最強模型的雲端 API 存取路徑出現主權性中斷風險。 - [科技新報, 2026-04] [來源]「Meta 員工自建 AI 用量排行榜秀 30 天就破 60 兆 Token，但資料外流迅速撤下」→ 觀察事實：企業內部 LLM 使用量規模龐大，且資料外流已成為實際發生的事件，非理論風險。 - [Android Authority, 2026-04] [來源]「Google Gemma 4 在您的口袋中：如何完全離線執行最新 AI」→ 觀察事實：可離線執行的高品質小模型已成為主流模型廠商的產品路線。 - [SemiEngineering, 2026-04] [來源]「邊緣 AI 無所不在、代理型、多模態且硬體加速」→ 觀察事實：產業共識已從「邊緣 AI 是否成立」轉為「邊緣 AI 如何硬體加速」。 - [The New Stack, 2026-06-15] [來源]「Anthropic 推 Claude Managed Agents，希望為您運營 AI Agent」→ 觀察事實：Agent 已從「工具」變成「需要被持續運營的資產」。 #### Step 2：條件變化分析 **改變的條件**：2026 年第二季前後，「企業可信任的最強 LLM 模型」的取得路徑從「穩定的雲端 API」變成「可能被地緣政治中斷的雲端 API」。同時，可在 8-16GB RAM 邊緣裝置執行的開源模型（Gemma 4、Mythos 等級替代品）品質達到「對企業有實用價值」的門檻。 - **時間點**：模型能力門檻在 2025 下半年至 2026 上半年達到實用線；地緣政治中斷風險在 2026-06 Anthropic 禁令事件後具體化。 - **改變類型**：同時是技術性（小模型能力跳升）＋監管性（AI 出口管制成為新工具）。 - **過去不存在**：兩年前（2024）開源小模型品質不足、且 AI 模型尚未被視為國家戰略管制品。 - [推斷] 標記：模型能力門檻達標屬 [來源]；管制工具化屬 [來源]；兩者交會時點為 [推斷]。 #### Step 3：趨勢導出 - **Q1 條件變化**：可信任 LLM 的供應路徑從「單一雲端 API」變為「多元化、含本地部署」。 - **Q2 傳導機制**：企業 CIO 在評估 AI 投資時，新增「供應中斷風險」維度 → 採購邏輯從「最強模型優先」轉為「可控性 × 能力」的二維評估 → 工作負載開始分流到邊緣/本地。 - **Q3 為何此時足夠**：在 2026-06 之前，禁令是理論風險；Anthropic 事件讓 CFO 級別人士（如加拿大總理卡尼公開警告）開始把這個風險寫進財報敘述。臨界點是「風險具象化」。 - **Q4 反事實**：如果 Anthropic 事件沒發生、且 Gemma 4 等級模型不存在，企業仍會繼續用雲端 API，趨勢不會發生。兩者缺一即斷。 ✅ **產業趨勢**：**LLM 推論工作負載的「主權化分流」——從純雲端集中走向「雲端＋區域＋裝置端」的分層架構，且分層的驅動力不只是延遲/成本，而是「可控性」。** 驗證： - ✅ 宏觀性：影響所有採用 LLM 的企業，不限單一公司 - ✅ 長期性：地緣政治競爭是 5-10 年級結構 - ✅ 不可逆性：即使禁令解除，企業已被教育「依賴單一供應商=風險」 #### Step 4：市場新機會浮現 - **Q1 競爭條件改變**：原本「邊緣 SoC 競爭」的維度是 TOPS/W、影像處理能力；新維度變成「能否在 8-32GB 記憶體預算內，以可接受功耗執行 7B-30B 參數 LLM，且支援 Agentic 工作流（多輪、長 context、tool calling）」。 - **Q2 傳導機制**：企業 IT 採購會把「本地 LLM 推論吞吐」寫入 RFP → 邊緣裝置 OEM（工業 PC、Mini PC、NAS、企業閘道）需要新的 SoC 規格 → 創造一個「介於手機 SoC 與資料中心 GPU 之間」的真空帶。 - **現有競爭者為何沒滿足**： - Qualcomm/MediaTek 的手機 SoC 記憶體頻寬上限（LPDDR5X ~68GB/s）對 13B+ 模型不夠 - NVIDIA Jetson 系列功耗（15-60W）與成本（>$500）對企業批量部署過高 - Intel/AMD x86 走通用路線，沒有針對 LLM 推論的記憶體頻寬最佳化 - 結構性原因：**利益衝突**——NVIDIA 的資料中心 GPU 才是金牛，做太強的邊緣方案會自我侵蝕 - **Q4 反事實**：如果主權化分流趨勢不存在，企業會繼續用雲端，這個 SoC 真空帶不會有實質需求。 ✅ **市場新機會**：**為「企業本地部署的 Agentic AI 工作站／閘道」設計專用 SoC，目標規格：支援 13B-30B 模型 INT4 推論、記憶體頻寬 100-200GB/s、功耗 25-50W、含 NPU + 安全飛地（Secure Enclave）。** #### Step 5：目標客群與 JTBD **問題 A：為什麼是 X，不是其他客群？** 候選客群： - ❌ **消費者（手機/PC）**：Apple、Qualcomm、MediaTek 已強勢卡位，且消費者支付意願綁定整機售價，難以擷取 SoC 端溢價。 - ❌ **超大規模資料中心**：NVIDIA H/B 系列、AWS Trainium 已主導，新進入者無生態。 - ✅ **企業本地部署裝置 OEM（工業電腦、企業閘道、產業專用伺服器廠商，如研華、凌華、神基、Supermicro 邊緣產品線）** **確立**：因此目標客群是企業邊緣裝置 ODM/OEM，因為他們**需要差異化規格但無能力自研 SoC**，且其客戶（製造業、醫療、金融分行）有明確「資料不能出廠」的合規需求。 **問題 B：結構性支付意願來源** - 工業電腦廠的毛利率（25-35%）來自「客製化規格 × 認證壁壘」，他們需要新的差異化點來維持毛利 - 他們的終端客戶（製造業 IT）在 2026 年首次面臨「老闆指定要部署 LLM Agent 但不能上雲」的具體 JTBD - 支付意願來自「無此晶片就接不到單」的存亡壓力，非「優化體驗」 **問題 C：有能力的廠商為何沒做？** - **選擇不做（利益衝突）**：NVIDIA 此產品線會侵蝕 Jetson Orin 與 DGX Spark 的定位 - **商業模式不符**：Qualcomm 以手機量為核心，企業邊緣量太小（單款 10-50 萬顆 vs 手機千萬顆）不值得專案立項 - **技術障礙較低**：對 MediaTek、聯發科旗下天璣產品線、或 Marvell 這類 ASIC 玩家是可行的，但**他們的組織尚未把這個市場列為 P0** ✅ **JTBD**：**「在客戶資料不離開廠區/分行的前提下，部署一個能執行多輪推理、調用內部 API、處理文件/影像/語音多模態的 AI Agent，且總體擁有成本（裝置 + 三年運維）低於同等雲端 API 費用」** #### Step 6：痛點 - **具體環節阻礙**：執行 13B+ 模型時，**記憶體頻寬不足導致 token 生成速度 < 10 tokens/sec**，無法支撐 Agent 的多輪工具呼叫（一次任務需數百到數千 token）。 - **現有方案為何無法改善**： - x86 CPU + 整合 GPU：頻寬 50-80GB/s，瓶頸在 memory wall - Jetson Orin：頻寬達 204GB/s 但價格 $1,999、功耗 60W，對企業批量部署的 TCO 不可接受 - 機制：**現有方案是「為其他主場景設計、順便做 LLM」**，不是為 LLM 主場景設計 - **Q4 反事實**：若 Jetson 降價到 $300 且功耗 25W，這個痛點消失。但 NVIDIA 因利益衝突不會這麼做。 - **痛點深度排序**：記憶體頻寬 > 功耗/散熱 > 軟體生態 > 安全飛地。前兩項是「不解決就無法商用」級。 ✅ **核心痛點**：**在 25-50W 功耗、$200-500 BOM 成本預算內，無法獲得支撐 13B+ LLM 實用推論速度的記憶體頻寬。** #### Step 7：客戶價值 **解決方案**：採用 LPDDR5X-on-package（類似 Apple M 系列 UMA 架構）或低成本 HBM 變體，搭配專用 LLM NPU（支援 INT4/FP4 動態切換、KV cache 壓縮硬體加速），達成 150GB/s 頻寬 × 30W 功耗 × $250 BOM 的 SoC。 - **機制**：透過 UMA 架構消除 CPU-GPU-NPU 資料搬移，KV cache 硬體壓縮減少 2-3x 記憶體佔用，使 13B 模型在 16GB 記憶體上以 >20 tokens/sec 運行 - **效益**：客戶 OEM 可推出單價 $1,500-2,500 的企業 AI 閘道（vs Jetson 整機 $4,000+），對終端企業 TCO 比雲端 API 低 40-60%（每月 $200 設備攤提 vs $500+ API 費用） - **成本**：客戶需重新設計散熱、認證、軟體 stack 整合（估 6-9 個月 NRE） - **效益 > 成本機制**：單一機種生命週期 3-5 年、10-50 萬顆出貨，NRE 攤提後每顆成本 <$5，相對於每顆 SoC 的 $250 BOM 邊際成本可忽略 - **競爭壁壘時間窗口**：[推斷] NVIDIA 若推出 Jetson Nano LLM 版需 18-24 個月（架構重設計）；Qualcomm/MediaTek 若立項需 24-30 個月。窗口寬度約 **18-30 個月**。 - **Q4 反事實**：若雲端 API 沒有被「資料主權」議題打擾，客戶會選雲端，解決方案無客戶價值。 #### Step 8：商業價值 **商業模式**：SoC 銷售 + Reference Design + 軟體開發套件（含模型量化工具鏈）授權，採「晶片+軟體」雙收入。 - **擷取機制**：SoC ASP $80-150（vs 手機 SoC $30-60）、軟體授權每裝置 $20-50/年（綁定 OTA 與安全更新） - **定價權來源**： - **架構壁壘**：UMA 設計需要 IP 整合能力，非純 RTL 拼裝 - **認證綁定**：工業/醫療客戶通過認證後切換成本含重新認證（6-12 個月、$50-200 萬） - 轉換成本組成：認證費用 + 軟體 stack 重寫（約 3-6 工程師月）+ 客戶端產品重設計 - **護城河動態**： - **加深機制**：每多一個 OEM 採用，模型量化工具鏈累積更多優化參數 → 後進者推論效率落後 20-30% - **削弱威脅**：NVIDIA 可能在 2027-2028 推出針對性產品；中國同類晶片（如愛芯元智、瑞芯微）以價格戰殺入 - **頂峰時間點**：[推斷] 2027 年底至 2028 年中，3-4 億顆累計出貨後護城河最深 - **商業價值上限**：[推斷] 若 18-30 個月窗口內取得 30% 企業邊緣 LLM 裝置市佔，年收入 5-15 億美元；窗口後若護城河消失但保留 10-15% 市佔，仍有 2-5 億美元年收 - **預估市場規模**：[未驗證，建議搜尋確認] 企業邊緣 AI 裝置 2027 年市場規模估 30-80 億美元（含整機） --- ### 機會 2：RISC-V 取代 ARM 的「特定控制器」市場——SSD 控制器、CPE/路由器、車用 MCU 的「去 ARM 化」 #### Step 1：事實觀察 - [科技新報, 2026-04] [來源]「三星 SSD 控制器導入 RISC-V，逐步降低對 ARM 依賴」→ 觀察事實：龍頭儲存廠正式量產 RISC-V 控制器。 - [Tom's Hardware, 2026-06-15] [來源]「中國最高法院禁止英飛凌在華銷售 GaN 功率芯片」→ 觀察事實：中國正在系統性使用司法工具排除特定外國半導體 IP。 - [Ars Technica, 2026-04] [來源]「FCC 路由器禁令包括便攜式熱點」+「FCC 豁免 Netgear 外國路由器禁令，未給出解釋」→ 觀察事實：美國 CPE 設備供應鏈進入「政治選擇性開放」階段。 - [SemiEngineering, 2026-04] [來源]「故障注入框架針對 RISC-V 安全弱點」→ 觀察事實：學界把 RISC-V 視為主流到值得攻擊研究的程度。 #### Step 2：條件變化分析 **改變的條件**：ARM 在 2022-2024 的授權費調漲＋軟銀回收 IPO 後的股東壓力，使 ARM 從「中立 IP 供應商」轉為「積極定價的營利公司」。同時 RISC-V 工具鏈（GCC、LLVM、商業 IP 如 SiFive、Andes）在 2024-2025 達到「對 SSD/MCU/網通應用足夠」的成熟度。 - **時間點**：成熟度交叉發生在 2025 年；三星量產是「第一個一線玩家公開承認跨越」的訊號 - **改變類型**：經濟性（ARM 費用）＋ 監管性（地緣政治推升 IP 主權考量） - **過去不存在**：2022 年前 RISC-V 工具鏈不成熟、且 ARM 費率仍可接受 - [來源] 標記：三星新聞為來源；ARM 漲價時間軸為 [推斷，建議搜尋確認] #### Step 3：趨勢導出 - **Q1**：「使用 ARM 的隱性成本」從「授權費」擴大為「授權費 + 地緣風險 + 客戶要求 IP 多元化」 - **Q2**：大型系統廠（三星、華為、潛在的 Apple/Google）開始把 RISC-V 列為「策略性第二來源」→ 推動 IP 供應商、EDA 工具、編譯器投資 RISC-V → 工具鏈良性循環 - **Q3 為何此時足夠**：三星 SSD 控制器是「年產數億顆」級別應用，首次證明 RISC-V 在大量產品線可商用，跨過「示範→主流」門檻 - **Q4 反事實**：若 ARM 維持原本費率＋RISC-V 工具鏈未成熟，趨勢不會發生 ✅ **產業趨勢**：**RISC-V 從「研究/低階 MCU」滲透到「中階控制器（SSD/網通/車用 MCU/IoT 閘道）」，但暫不挑戰高階 AP/手機 SoC。** #### Step 4：市場新機會浮現 - **Q1 競爭條件改變**：原本 SSD/CPE/車用 MCU 的競爭維度是「性能 × 成本」；新增「IP 自主性」維度 - **Q2 機制**：客戶 RFP 開始要求「ARM 替代方案路線圖」→ 製造商需要 RISC-V 版本 SKU → 創造 RISC-V IP 與週邊（debug、安全、AI 加速擴充）需求 - **現有競爭者為何沒滿足**： - ARM 本身：利益衝突，不會推 RISC-V - SiFive、Andes：聚焦 CPU core，缺乏完整 SoC 平台（含 PHY、安全、AI 加速） - **結構性原因**：RISC-V 生態的「整合層」是空的——有 core、有編譯器，但缺「現成可用的應用導向參考設計」 - **Q4 反事實**：若無地緣政治壓力，客戶會繼續用 ARM 不費事 ✅ **市場新機會**：**提供針對「SSD 控制器、CPE/家用閘道、車用 MCU」應用的 RISC-V SoC 平台與參考設計（含驗證過的安全、AI 推論小核、業界連接介面），讓系統廠以最低 NRE 切換。** #### Step 5：目標客群與 JTBD **問題 A：為什麼是這個客群？** 候選： - ❌ **手機 AP 客戶**：軟體生態（Android NDK）強綁 ARM，切換成本太高，至少 5 年不會動 - ❌ **資料中心 CPU**：Intel/AMD/ARM Neoverse 已三強競爭，RISC-V 還太弱 - ✅ **控制器類 ODM/OEM（SSD 模組廠如群聯/慧榮、CPE 廠如智易/中磊、車用 Tier 1）** **確立**：因為這些客戶的軟體 stack 主要是 firmware（C/RTOS），不依賴 Android 生態，**架構切換成本只在硬體驗證**而非軟體生態。 **問題 B：結構性支付意願** - SSD 控制器毛利率薄（10-15%），ARM 授權費佔 BOM 1-3% 是真實的痛 - CPE 廠面臨美/中雙重審查，需要「ARM 版」與「非 ARM 版」並行 - 支付意願來自「降本」＋「不切換就失去某些區域市場」 **問題 C：有能力者為何沒做？** - SiFive 等 IP 廠：**商業模式不符**——他們賣 IP 不賣 SoC 平台 - 大型 SoC 廠（聯發科、瑞昱）：**選擇不做**——既有 ARM 產品線正在賺錢，主動切換有自我侵蝕風險 - 中國廠商（瑞芯微、全志）：正在做但**生態與信任度**對國際客戶不夠 ✅ **JTBD**：**「在 12-18 個月內推出 ARM 版產品的 RISC-V 等效 SKU，且性能不退步、軟體移植 NRE < 既有產品開發成本的 30%」** #### Step 6：痛點 - **具體環節阻礙**：RISC-V core 可選但**周邊 IP（PCIe、USB、安全模組、AI 小核）整合驗證**沒有 turnkey 方案，每家系統廠都要自己整合 -
