# SoC 策略分析報告 ## 資料區間：2026-04-27 ~ 04-29 --- ## 任務一：結構性機會偵測 --- ### 🎯 機會 1：邊緣裝置上的「離線可用 Agentic AI」推論 SoC #### 🔵 啟動層：產業趨勢 ──→ 市場新機會 **產業趨勢**：AI 從「雲端集中式推論」轉向「邊緣 + 多模態 + 代理化（Agentic）+ 硬體加速」的分散式架構。 - ✅ 宏觀：影響整個半導體與裝置產業 [來源：SemiEngineering「邊緣 AI 無所不在、代理型、多模態且硬體加速」, 2026.04] - ✅ 長期：模型小型化（Gemma 4 可離線執行）+ 隱私法規 + 雲端成本壓力共同驅動 [推斷] - ⚠️ 不可逆性：[推斷] — 雲端成本若大幅下降可能延緩，但隱私與延遲需求是結構性的 **趨勢佐證**： - [來源：Android Authority「Google Gemma 4 在您的口袋中：完全離線執行最新 AI」, 2026.04] - [來源：科技新報「Meta 員工 30 天破 60 兆 Token」, 2026.04] — 雲端 token 消耗爆炸性成長，**機制**：當推論成本 × 使用頻次超過邊緣硬體攤提成本時，邊緣化變成經濟必然 [推斷] - [來源：SemiEngineering, 2026.04] **機會邊界**：高階智慧手機、AI PC、車用 HMI、企業端裝置（會議室／POS／工控 HMI）上，能執行「多步驟代理任務」（不只單次 inference）的 SoC NPU 子系統。 **因果回答（趨勢→機會的打破平衡機制）**： 雲端 LLM 規模擴張 → token 成本與延遲成為瓶頸 → 模型蒸餾與量化技術成熟（Gemma 4 等）→ 原本「必須上雲」的 AI 任務在裝置端變得可行 → 打破「邊緣 SoC 只做感知、雲端做推理」的舊平衡 → 浮現「邊緣 Agentic NPU」這個新戰場 [推斷] #### 🟡 鎖定層：市場新機會 ──→ 痛點 **目標客群與 JTBD**： - 客群：手機／PC OEM（Samsung、Lenovo、Dell）、車廠 Tier-1 - JTBD：在裝置上執行多步驟代理任務（讀取本地檔案 → 推理 → 呼叫 API → 產生結果），且滿足隱私、延遲、續航三項約束 **因果回答（具體阻礙的機制）**： 現有方案（Qualcomm Hexagon NPU、MediaTek APU、Apple Neural Engine）主要為**單次 inference** 優化，但 agentic workflow 是**多輪、長 context、頻繁 KV-cache 讀寫**的工作負載 [推斷]。具體阻礙： 1. **記憶體頻寬瓶頸**：Agent 多輪推理需反覆存取 KV-cache，現有 NPU 的 LPDDR 介面頻寬不足 [推斷] 2. **異質排程缺位**：CPU/GPU/NPU 在多步驟任務中的切換成本高 [推斷] 3. **功耗曲線**：持續 agent 運作 vs 一次性 inference 的功耗模型不同，現有 SoC 未針對前者優化 [未驗證] 支付意願：[未驗證] — 需要訪談 OEM PM 確認；但從 Apple 不惜延後 Siri 上市可推斷 [推斷]，OEM 對「可用的裝置端 AI」支付意願極高。 #### 🟢 轉換層：痛點 ──→ 客戶價值 **解決方案**：針對 agentic workload 優化的 NPU 子系統 — 大容量 on-die SRAM、高頻寬 LPDDR5X/6、CPU-NPU 共享記憶體、低功耗持續運轉模式。 **因果回答**： - 效益：OEM 可推出「離線可用 AI 助理」差異化功能，避開雲端成本（每用戶每月可省 $X，[未驗證]） - 成本：BOM 增加（更大 SRAM、更快記憶體介面）、軟體 stack 重寫 - 效益 > 成本機制：當「離線 AI」成為手機規格戰必備項目時，OEM 沒有不買的選擇 [推斷] **為何現有廠商目前無法提供同等價值**： - Qualcomm/MediaTek 的 NPU 架構是 2-3 年前定案，難以快速調整 [推斷] - Apple 自用不外賣 - **時間窗口**：[推斷] — 約 12-18 個月，下一代旗艦 SoC 定案前 #### 🔴 收成層：客戶價值 ──→ 商業價值 **商業模式**：IP 授權 + 完整 SoC 銷售雙軌 **因果回答（價值擷取機制）**： - 定價權來源：① 架構壁壘（agentic NPU 的微架構需 2-3 年累積）② 軟體生態綁定（編譯器、模型 zoo）③ OEM 一旦選定難以中途換廠 - 護城河類型：架構 + 生態雙重；隨時間**加深**（軟體生態網路效應） - 風險：[未驗證] — 若 Google／Apple 開源其 NPU 編譯器，護城河會變淺 **市場規模與時間窗口**：[時間不明，建議搜尋確認] — 邊緣 AI SoC TAM 估計需引用 IDC/Gartner 最新報告 --- ### 🎯 機會 2：RISC-V 進入「儲存／網通控制器」的非 ARM 替代市場 #### 🔵 啟動層：產業趨勢 ──→ 市場新機會 **產業趨勢**：大型系統廠（三星、Google TPU、AWS Trainium）為降低 ARM 授權與供應風險，加速採用 RISC-V 於非主 CPU 場景。 - ✅ 宏觀：影響整個 IP 授權產業 - ✅ 長期：地緣政治 + 授權成本驅動 [推斷] - ✅ 不可逆：投入已沉沒（toolchain、ISA 擴展、Linux 支援）[推斷] **趨勢佐證**： - [來源：科技新報「三星 SSD 控制器導入 RISC-V，逐步降低對 ARM 依賴」, 2026.04] - [來源：Tom's Hardware「Google TPU V8 雙晶片設計」, 2026.04]（雖未明說 RISC-V，但 TPU 內部控制核心採 RISC-V 已是公開資訊 [來源]） **機會邊界**：SSD/NVMe 控制器、5G CPE 內的 baseband 周邊控制核、AI 加速器內的 host CPU、智慧電表／工控 — 不需要 Android 生態相容性的場景。 **因果回答**： ARM 授權費上漲 + 中美科技戰 → 大廠尋求「非主 CPU 但需可程式化」場景的替代 ISA → RISC-V toolchain 與 IP 生態成熟（SiFive、Andes、阿里平頭哥）→ 打破「ARM 在嵌入式控制器壟斷」的舊平衡 [推斷] #### 🟡 鎖定層：市場新機會 ──→ 痛點 **目標客群與 JTBD**： - 客群：儲存控制器廠（Phison、Silicon Motion）、網通晶片廠、AI 加速器設計公司 - JTBD：在不犧牲性能的前提下，將控制核從 ARM Cortex-R/M 替換為 RISC-V **因果回答（具體阻礙）**： - 現有 RISC-V IP 在**即時性（real-time）認證**、**功能安全（ISO 26262）**上仍弱於 ARM Cortex-R [未驗證] - 工具鏈（debugger、profiler）成熟度不足 [推斷] - 客製化 ISA 擴展碎片化 → 軟體可攜性差 [推斷] #### 🟢 轉換層：痛點 ──→ 客戶價值 **解決方案**：提供針對特定 vertical（儲存／網通）優化、含完整功能安全認證的 RISC-V 控制器 IP + toolchain。 **因果回答**： - 效益：節省 ARM 授權金 + 可客製 ISA（針對 SSD ECC、網路封包處理）+ 規避地緣風險 - 成本：軟體移植、認證重做 - 三星已選擇導入，證明效益 > 成本 [來源：科技新報, 2026.04] #### 🔴 收成層：客戶價值 ──→ 商業價值 **商業模式**：IP 授權（SiFive 模式）或控制器晶片直銷 **因果回答**： - 定價權：客製 ISA 擴展形成綁定（一旦客戶在自己 ISA 擴展上寫了韌體，難以更換 IP 廠） - 護城河：中等深度，會隨 RISC-V 生態標準化而部分稀釋 [推斷] - 風險：[未驗證] — 若 RISC-V International 強推標準擴展集，差異化空間縮小 **市場規模**：[時間不明，建議搜尋確認] --- ### 🎯 機會 3：FCC 路由器禁令下的「在地化 5G FWA / CPE SoC」 #### 🔵 啟動層：產業趨勢 ──→ 市場新機會 **產業趨勢**：美國對中國網通設備的法規圍堵從「品牌禁令」擴大到「特定 SKU 與便攜熱點」。 - ✅ 宏觀：影響全球網通供應鏈 - ⚠️ 長期：[推斷] — 取決於政治週期，但短期 3-5 用戶」, 2026.04] **機會邊界**：北美市場的 5G FWA CPE、便攜熱點、企業 SD-WAN 邊緣設備 — 排除中國品牌與部分中國 SoC 後的供應缺口。 **因果回答**： FCC 擴大禁令 → 北美 OEM（Netgear、Inseego、Cradlepoint）需要替代供應鏈 → Qualcomm 為主導但價格高 → 浮現「非中國、非 Qualcomm」的第三選擇空間 [推斷] #### 🟡 鎖定層：市場新機會 ──→ 痛點 **目標客群與 JTBD**：北美 CPE OEM，需在 18 個月內推出符合 FCC 規範的新品。 **因果回答（具體阻礙）**： - Qualcomm 5G modem 售價高、talk power 大 [推斷] - MediaTek 在北美 CPE 認證與電信商通過名單上覆蓋不足 [未驗證] - 中國 SoC（紫光展銳）受制裁無法用 [來源：FCC 禁令] 支付意願：高 — Verizon FWA 用戶 600 萬持續成長 [來源：Light Reading, 2026.04]，OEM 急需貨源。 #### 🟢 轉換層：痛點 ──→ 客戶價值 **解決方案**：5G NR Sub-6 + Wi-Fi 7 整合 SoC，已通過 FCC、Verizon／T-Mobile 認證，BOM 較 Qualcomm 低 20-30%。 **因果回答**： - 效益：OEM 取得認證合規、成本下降的雙重價值 - 成本：軟體移植（從 Qualcomm SDK 遷移） - 為何現有廠商無法：MediaTek 不熟北美電信商認證流程；認證需 12-18 個月 [推斷] — 形成時間壁壘 #### 🔴 收成層：客戶價值 ──→ 商業價值 **商業模式**：晶片直銷 + 參考設計 **因果回答**： - 定價權：認證壁壘（一旦通過 Verizon AVL 進入合格供應商名單，新進者需重新認證） - 護城河：中等，**會隨時間加深**（更多 OEM 設計綁定） - 風險：[未驗證] — 若 Qualcomm 大降價或川普政府放鬆禁令，機會窗口縮短 **市場規模**：[時間不明，建議搜尋確認] — 北美 FWA CPE 出貨量需引用 Dell'Oro 報告 --- ## 任務二：矛盾點警告 ### ⚠️ 矛盾 1：海外
